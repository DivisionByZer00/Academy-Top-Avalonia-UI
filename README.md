# Academy-Top-Avalonia-UI

#### Решения домашних заданий "Разработка кроссплатформенных GUI-приложений с Avalonia UI"

## Homework 01

### Тема: "Анатомия GUI приложения. Введение в Avalonia UI"

**Задание**: Создайте новый проект Avalonia с названием **Homework1_AboutMe**. Приложение должно содержать одно окно со следующими элементами:

**Заголовок окна:** «Портфолио: [Ваша Фамилия]».

**Размер окна:** Окно должно быть компактным (примерно 300x450) и фиксированным (запретить растягивание CanResize=»False»).

**Содержимое окна** (внутри StackPanel):
- *Заголовок:* Ваше имя (Крупный шрифт, выравнивание по центру).
- *Описание:* Краткий текст о себе (например: «Студент 3 курса, люблю C# и котиков»). Текст должен переноситься на новую строку, если он длинный (используйте свойство TextWrapping=»Wrap»).
- *Секретная информация:* TextBlock, который изначально скрыт (свойство IsVisible=»False»).
- *Панель кнопок*: Две кнопки, расположенные внизу.

**Логика кнопок**

**1. Кнопка «Показать секрет»:**      
   а. При нажатии скрытый текст становится видимым;      
   б. В тексте может быть интересный факт о вас;      
   в. *Подсказка:* Чтобы обратиться к тексту из C#, не забудьте дать ему имя (x:Name). В коде используйте свойство .IsVisible = true;.      
**2. Кнопка «Выход»:**      
   а. При нажатии приложение должно закрываться:      
   б. *Подсказка:* Для закрытия текущего окна используйте команду this.Close(); в C# коде.

## Homework 02

### Тема: "Интерактив и ввод данных. Элементы управления"

**Задание**: Разработайте приложение «Форма заказа пиццы».  
Представьте, что вы пишете интерфейс для терминала самообслуживания в пиццерии. Пользователь должен ввести свои данные, выбрать пиццу, настроить количество и подтвердить заказ.

**Требования к окну:**
- *Заголовок:* «PizzaOrder: [Ваша Фамилия]»
- *Размеры:* Фиксированные (например, 400x600), CanResize=»False».
- *Фон:* Светлый (например, White или Beige).

**Содержимое окна:**
- Блок «Клиент»;
- Блок «Выбор пиццы» (Один из вариантов);
- Блок «Добавки» (Множественный выбор);
- Блок «Количество»;
- Блок «Итог».

**Подсказки:**
- Для проверки пустоты строки используйте: string.IsNullOrWhiteSpace(MyTextBox.Text)
- Помните, что NumericUpDown.Value и CheckBox.IsChecked могут быть null, поэтому используйте безопасное приведение или проверку (например, CheckCheese.IsChecked == true).
- Чтобы сделать отступы между блоками, используйте свойство Margin у элементов или оберните группы элементов в отдельные StackPanel.

## Homework 03

### Тема: "Мастер компоновки"

**Задание**: Разработайте приложение «Карточка сотрудника».      
Вам нужно сверстать окно, которое выглядит как анкета сотрудника в корпоративной системе. Главное условие — использовать контейнер Grid для управления макетом.

**Требования к окну:**
- *Заголовок:* «Сотрудник: [Ваша Фамилия]»
- *Размеры:* Фиксированные (например, 500x350), CanResize=»False».
- *Фон:* Светло-серый (например, WhiteSmoke или #F0F0F0).

**Структура окна:**
Корневой элемент должен быть Grid. Вам нужно «нарезать» его следующим образом:
- Столбцы (Columns);
- Строки (Rows).

**Содержимое окна:**
- Фото сотрудника (Левый столбец);
- Имя сотрудника (Правый столбец, Верхняя строка);
- Данные анкеты (Правый столбец, Средняя строка);
- Кнопки управления (Нижняя строка).

## Homework 04

### Тема: "Стили и ресурсы"

**Задание**: Разработайте приложение «Профиль + Тема».      
Нужно сверстать окно личного кабинета с карточкой профиля и блоком оформления. Главное условие — использовать ресурсы и стили (Styles) так, чтобы внешний вид можно было менять централизованно.

**Требования к окну:**
- Заголовок: «Профиль: [Ваша Фамилия]».
- Размеры: фиксированные (например, 520x420), CanResize=»False».
- Поддержка двух тем: Light и Dark (переключение кнопкой или ToggleSwitch).

**Требования к ресурсам и стилям:**
- Создайте минимум 6 ресурсов (например: AppBackground, CardBackground, TextBrush, SubTextBrush, AccentBrush, SpacingM).
- Используйте DynamicResource для значений, которые должны меняться при переключении темы.
- Создайте минимум 3 стиля: базовый для Button, стиль Button.primary и стиль для заголовка TextBlock.h1 (через Classes).
- Используйте минимум 2 псевдокласса (например: :pointerover, :pressed, :disabled).

**Содержимое окна:**
- Заголовок страницы (например, «Личный кабинет»).
- Карточка «Профиль» (Border): имя, email, город (TextBox или ComboBox).
- Карточка «Оформление»: переключатель темы и две кнопки (Сохранить/Отмена), одна из которых — primary.
- Используйте отступы и размеры через ресурсы, а не числа «вручную» в каждом элементе.

## Homework 05

### Тема: "Data Binding (привязки данных)ы"

**Задание**: Разработайте приложение «Профиль пользователя с живым превью».      
*Идея:* слева пользователь заполняет форму, справа сразу же отображается карточка-превью. Все изменения должны появляться автоматически через Binding, без ручных присваиваний TextBlock.Text в обработчиках.

**Требования к окну:**
- *Заголовок:* «ProfileCard: [Ваша Фамилия]»
- *Размеры:* фиксированные (например, 650x420), CanResize=»False».
- *Макет:* корневой элемент Grid с двумя колонками (форма / превью).

**Поля формы (минимум 6 привязок):**
- ФИО (TextBox → FullName).
- Роль (ComboBox → SelectedRole). Например: «Разработчик», «Дизайнер», «Тестировщик», «Менеджер».
- Email (TextBox → Email).
- Телефон (TextBox → Phone).
- Стаж (NumericUpDown → ExperienceYears).
- Подписка на новости (CheckBox → IsSubscribed).

**Карточка‑превью справа должна показывать:**
- Большой заголовок: FullName.
- Роль и стаж в одной строке (можно через вычисляемое свойство SummaryLine).
- Email и телефон.
- Предупреждение: если Email пустой — вывести текст «Email не указан» красным.

**Обязательные условия по технике (проверка будет по этим пунктам):**
- DataContext задаётся один раз в MainWindow.axaml.cs (например, DataContext = new ProfileViewModel()).
- ProfileViewModel реализует INotifyPropertyChanged.
- Для зависимых/вычисляемых свойств (SummaryLine, WarningText) вызывается PropertyChanged при изменении исходных данных.
- Не используйте обработчики TextChanged/ValueChanged для обновления превью (они не нужны).

**Условие по теме прошлого урока (ресурсы и стили):**
- Добавьте минимум 3 ресурса (например кисти/цвета/отступы) в Window.Resources или App.axaml.
- Используйте минимум 2 стиля (например для TextBlock-меток слева и для Border карточки справа).

**Подсказки:**
- Для ComboBox можно задать Items в XAML (несколько ComboBoxItem) и привязать SelectedItem к свойству ViewModel.
- Если WarningText — пустая строка, TextBlock может оставаться видимым: он просто не покажет текст. (Скрытие через конвертер/стиль — бонус.)
