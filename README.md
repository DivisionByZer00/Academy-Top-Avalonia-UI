**Задание:** Разработайте приложение «Профиль пользователя с живым превью».      
***Идея:*** слева пользователь заполняет форму, справа сразу же отображается карточка-превью. Все изменения должны появляться автоматически через Binding, без ручных присваиваний TextBlock.Text в обработчиках.

**Требования к окну:**
- ***Заголовок:*** «ProfileCard: [Ваша Фамилия]»
- ***Размеры:*** фиксированные (например, 650x420), CanResize=»False».
- ***Макет:*** корневой элемент Grid с двумя колонками (форма / превью).

**Поля формы (минимум 6 привязок):**
- ФИО (TextBox → FullName).
- Роль (ComboBox → SelectedRole). Например: «Разработчик», «Дизайнер», «Тестировщик», «Менеджер».
- Email (TextBox → Email).
- Телефон (TextBox → Phone).
- Стаж (NumericUpDown → ExperienceYears).
- Подписка на новости (CheckBox → IsSubscribed).

**Карточка‑превью справа должна показывать:**
- Большой заголовок: FullName.
- Роль и стаж в одной строке (можно через вычисляемое свойство SummaryLine).

**Обязательные условия по технике (проверка будет по этим пунктам):**
- DataContext задаётся один раз в MainWindow.axaml.cs (например, DataContext = new ProfileViewModel()).
- ProfileViewModel реализует INotifyPropertyChanged.
- Для зависимых/вычисляемых свойств (SummaryLine, WarningText) вызывается PropertyChanged при изменении исходных данных.
- Не используйте обработчики TextChanged/ValueChanged для обновления превью (они не нужны).

**Условие по теме прошлого урока (ресурсы и стили):**
- Добавьте минимум 3 ресурса (например кисти/цвета/отступы) в Window.Resources или App.axaml.
- Используйте минимум 2 стиля (например для TextBlock-меток слева и для Border карточки справа).

**Подсказки:**
- Для ComboBox можно задать Items в XAML (несколько ComboBoxItem) и привязать SelectedItem к свойству ViewModel.
- Если WarningText — пустая строка, TextBlock может оставаться видимым: он просто не покажет текст. (Скрытие через конвертер/стиль — бонус.)

**Задание:** Разработайте приложение «Калькулятор чаевых (MVVM)».      
Это небольшое приложение должно считать чаевые и итоговую сумму «на человека». Главное требование: никакой логики в MainWindow.axaml.cs кроме назначения DataContext. Никаких обработчиков Click.

- Поле «Количество людей» (целое число, минимум 1).

- Кнопка «Добавить» реализована через Command. Кнопка должна быть неактивной, если форма невалидна (IsEnabled через Binding).

**Технические требования:**
- Структура проекта: папки Views (если используете), Controls (компоненты), ViewModels, Models.
- Код-стиль: понятные имена, без «magic numbers» в логике.
- В XAML избегайте жёстких цветов: по возможности используйте ресурсы/стили из проекта.
- Компоненты должны переиспользоваться: изменение разметки компонента должно отражаться во всех местах, где он используется.
